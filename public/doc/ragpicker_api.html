NOTE:

  • TEST DOMAIN: https://ragpickers.herokuapp.com/
  • API TEST TOOL: POSTMAN / CURL COMMAND 
  • API FOR USER SCREEN: /api/user/*
  • API FOR RAGPICKER SCREEN: /api/picker/*
  • EVERY REQUEST NEEDS TO ADD ‘1234512345’ TO HEADER AS ‘x-svalue-auth’ (TEST/STAGING ENV).
  • HTTP CODE RESPONSE:
  • 200: success, and client need to update data
  • 204: success, client doesn’t need to update data.
  • 20x: success request
  • 400: bad request, client post wrong data format, and malformed request syntax
  • 401: unauthorized request (x-svalue-auth is wrong)
  • 403: invalid request/action that is not allowed
  • 404: request for missing object that is not exisited in system
  • 405: could not process the action because of requesting with wrong data
  • 40X: client error
  • 500: server got an exception. Request server admin for details

  • DATA SUCCESS FORMAT: json of object or array
  • DATA ERROR FORMAT: {error: {msg: “MESSAGE HERE”, code: “HANDLE CODE”}}
  • TEST GCM SENDER ID: 951950624682
  • PICKUP STATUS:
  • pending: just created by owner, has not accepted yet
  • accepted: has been accepted by ragpicker
  • done: proceeded and done by ragpicker
  • canceled: has been canceled by owner
  • expired: no one picked
